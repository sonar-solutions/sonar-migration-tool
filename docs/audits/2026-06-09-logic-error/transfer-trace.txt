NOTES: SCOPE: phases 2 (structure) and 3 (mappings) only. Both are pure local JSON->CSV transforms; NO HTTP/API calls. defaultOrganization PRE-POPULATION: RunStructure forwards cfg.defaultOrganization as the variadic defaultOrgKey into MapOrganizationStructure, which writes SonarCloudOrgKey into EVERY org row, so organizations.csv sonarcloud_org_key is pre-filled. Consequence (transfer.go:447-452): runTransferMigrate leaves migrate DefaultOrganization UNSET to avoid the 'mapping defined, default ignored' WARN. RunMappings is NOT given defaultOrganization. ORDERING in RunMappings is enforced by Go data dependencies (MapGroups takes profiles+templates), NOT by the randomized exports map iteration. UNIQUE-KEY scheme: serverURL+entityKey join keys throughout (projectOrgMapping = serverURL+projectKey -> sonarqube_org_key which equals the project's uniqueBindingKey). GUARDS: both phases error on empty extract mapping; RunMappings also errors if projects.csv has 0 rows.

- emitReports passes exportDir as BOTH outputDir and exportDir: summary.GenerateReports(runDir, exportDir, exportDir). So both the PDF and the markdown are written to the TOP-LEVEL export dir (cfg.exportDir), NOT inside runDir. runDir = exportDir/<runID> is only the SOURCE of the JSONL/log data.
- File outputs are fixed names: <exportDir>/migration_summary.pdf and <exportDir>/migration_summary.md, mode 0o644. A prior run's files are overwritten (os.WriteFile truncates).
